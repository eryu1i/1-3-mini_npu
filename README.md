## 결과 리포트

### 실패 원인 분석

테스트 결과 총 3가지 유형의 실패 케이스가 발생했다.

**1. 판정 불일치 (size_5_2)**
Cross 패턴을 입력했으나 expected를 X로 설정한 케이스다. MAC 연산 결과 Cross 점수가 높게 나와 Cross로 판정됐으나 expected가 X이므로 FAIL 처리됐다. 이는 데이터 문제로, 로직 자체는 정상 동작한다.

**2. 동점으로 판정 불가 (size_5_3)**
전부 0으로 이루어진 패턴을 입력한 케이스다. Cross 필터와 X 필터 모두 0점이 나와 두 점수 차이가 epsilon(1e-9) 미만이므로 UNDECIDED로 판정됐다. expected가 Cross 또는 X 중 하나이므로 항상 FAIL이 된다.

**3. 스키마 오류 (size_5_4)**
size_5 패턴 키에 4×4 배열을 입력한 케이스다. 스키마 검증 단계에서 크기 불일치가 감지되어 MAC 연산 없이 FAIL 처리됐다. 프로그램은 해당 케이스를 건너뛰고 계속 실행된다.

### 시간 복잡도 분석

MAC 연산의 시간 복잡도는 O(N²)이다. N×N 크기의 패턴과 필터를 처리할 때 모든 위치를 순회하므로 총 N²번의 곱셈과 누적합 연산이 필요하다. 크기가 2배 증가하면 연산 횟수는 4배 증가한다. 예를 들어 5×5는 25번, 13×13은 169번, 25×25는 625번의 연산이 필요하다. 실제 AI 모델은 수백×수백 크기의 필터를 수천 개 사용하므로 연산량이 기하급수적으로 증가하고, 이를 병렬 처리하기 위해 NPU가 필요하다.

### 부동소수점 및 비교 정책

점수 비교 시 단순 == 비교 대신 epsilon(1e-9) 기반 비교 정책을 적용했다. 실제 AI에서는 필터값이 실수이므로 부동소수점 오차가 발생할 수 있어 두 점수 차이가 1e-9 미만이면 동점으로 간주하고 UNDECIDED로 처리한다.
